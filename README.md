Fluxo TDD para Validação de Senhas

Este repositório apresenta a aplicação da metodologia TDD (Test-Driven Development) no desenvolvimento de um módulo de validação de senhas utilizado em um contexto de fintech brasileira. O processo foi conduzido seguindo o ciclo RED → GREEN → REFACTOR, garantindo que cada requisito fosse inicialmente representado por um teste antes da implementação do código.

Funcionalidades Implementadas

O módulo contempla as seguintes regras de validação:

Senha com no mínimo 8 caracteres;
Presença obrigatória de pelo menos uma letra maiúscula;
Presença obrigatória de pelo menos uma letra minúscula;
Inclusão de ao menos um número;
Inclusão de pelo menos um caractere especial dentre os seguintes: !@#$%^&*;
Proibição de espaços em branco na senha.
